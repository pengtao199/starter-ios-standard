# Docs Index

`Docs/` 用来放项目说明、PRD、调研材料、接口参考和非规则类文档。

这里不是开发规则入口。开发规则统一看根目录 `AGENTS.md` 和 `Rules/`；阶段计划、执行方案和决策记录统一放 `Plans/`。

## 文档边界

1. 产品说明、PRD、需求背景放 `Docs/`
2. 开发规则、目录约束、编码约束放 `AGENTS.md` 和 `Rules/`
3. 阶段计划、任务拆解、执行记录放 `Plans/`
4. Feature / Page / Paywall 起手模板放 `Templates/`

## 命名规范

文档文件使用两位序号开头，后接英文大写短横线名：

```text
00-XXXX.md
01-XXXX.md
02-XXXX.md
```

规则：

1. 序号表示创建顺序，不表示优先级
2. `XXXX` 使用英文大写短横线命名
3. 一个文档只覆盖一个明确主题
4. 稳定规则不要沉淀在 `Docs/`，应迁移到 `Rules/`
