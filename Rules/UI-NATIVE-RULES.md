# UI Native Rules

## Scope

适用于 iOS UI、交互控件、导航、系统能力入口和视觉重构。

## Goal

优先使用 iOS 原生体验，保持 Starter 稳定、熟悉、可维护。

## Rules

- SwiftUI 原生控件优先，例如 `NavigationStack`、`TabView`、`List`、`Form`、`Button`、`Picker`、`Sheet`。
- 系统图标优先使用 SF Symbols。
- 设置、订阅、法务、分享、反馈等系统路径优先遵循 iOS 常见交互。
- 颜色、背景和列表层级优先使用系统语义色。
- 自定义控件必须服务明确功能，不为炫技或装饰牺牲可读性。
- 动效只用于反馈状态变化，不阻塞操作路径。
- 可访问性、动态字体和深浅色模式必须随原生控件自然工作。

## Avoid

- 自定义 Tab、导航栏、弹窗来替代系统控件，除非产品需求明确且收益足够。
- 过度玻璃、渐变、动画或复杂装饰。
- 用图片或自绘元素替代可访问的文本与按钮。
- 让视觉层承担业务状态逻辑。

## Checklist

- 是否存在可直接使用的系统控件。
- 自定义 UI 是否有明确功能理由。
- 深浅色、动态字体、VoiceOver 是否没有被破坏。
- 交互是否符合 iOS 用户预期。
